# European Union Artificial Intelligence Act Compliance Checklist (February 20, 2026)

This checklist is aligned to Regulation (EU) 2024/1689 and the current timeline status on February 20, 2026.

## Regulatory timeline status

- [ ] Confirm entry into force date recorded as August 1, 2024.
- [ ] Confirm six month obligations recorded as active from February 2, 2025.
- [ ] Confirm twelve month obligations recorded as active from August 2, 2025.
- [ ] Confirm twenty four month obligations are planned for August 2, 2026.
- [ ] Confirm thirty six month obligations are planned for August 2, 2027.

## System classification and role

- [ ] Confirm intended purpose is documented.
- [ ] Confirm prohibited practice screening is documented.
- [ ] Confirm risk category is documented and approved.
- [ ] Confirm operator role is documented for provider responsibilities.
- [ ] Confirm operator role is documented for deployer responsibilities.

## Baseline obligations that apply now

- [ ] Artificial intelligence literacy records are maintained.
- [ ] User transparency notice is visible in the user interface.
- [ ] Incident and serious event handling process is documented.
- [ ] Access control is enabled in production.
- [ ] Request logging and traceability are retained.

## If system is classified as high risk

- [ ] Risk management system completed.
- [ ] Data governance documentation completed.
- [ ] Technical documentation completed.
- [ ] Record keeping controls completed.
- [ ] Human oversight controls completed.
- [ ] Accuracy robustness and security controls completed.
- [ ] Quality management system completed.
- [ ] Post market monitoring plan completed.
- [ ] Fundamental rights impact assessment completed for deployer context.

## Release gate

- [ ] Run compliance readiness command and confirm pass.
- [ ] Run full stable test command and confirm no failures.
- [ ] Keep signed approval from engineering, operations, and legal owners.
